# 能力检查

## 实体

**函数**是基本实体。每个函数有一个**名字**，名字中编码了该函数的**能力集合**。

## 能力

能力是函数的运行时性质的声明，共有八种：

| 能力 | 含义 |
|------|------|
| A | 异步执行 |
| B | 可能阻塞线程 |
| I | 执行 I/O 操作 |
| M | 修改共享可变状态 |
| P | 可能 panic |
| S | 有副作用（读写全局、环境变量、随机等） |
| T | 依赖线程局部状态 |
| U | 包含不安全操作 |

> 历史遗迹：早期设计曾有能力 `E`（可能返回错误），现已废弃——由 `Result` / `Option`
> 这类类型系统机制承担，不需要在函数名中另行标注。

## 命名约定

函数名以 `rvs_` 开头。名字的最后一节（以下划线分隔）若全由能力字母组成，则视为能力后缀。

例：
- `rvs_add` → 无能力
- `rvs_divide_P` → 能力 {P}
- `rvs_write_db_ABI` → 能力 {A, B, I}

## 调用规则

函数 A 调用函数 B 时，A 的能力集必须是 B 的能力集的超集。

换言之：B 有的每一个能力，A 都必须有。A 有的而 B 没有的，无妨。

**"我有，方可调你。"**

违反此规则的调用即为**违规**。

## 处理流程

1. 扫描所有 `.rs` 源文件
2. 识别所有以 `rvs_` 开头的函数定义
3. 从函数名中萃取能力集合
4. 在函数体中识别所有对 `rvs_` 函数的调用（包括直接调用和方法调用）
5. 对每对调用关系检查能力合规性
6. 报告所有违规
